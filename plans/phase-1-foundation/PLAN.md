# Phase 1: Foundation, Design System, Auth & Convex Setup

## Overview

This phase builds the entire app foundation: project structure, design system, local storage (MMKV + Zustand), Convex backend with Better Auth (Google, Apple, Phone OTP), biometric/PIN local app lock, onboarding flow, and the tab-based navigation shell. Every subsequent phase depends on this.

## Prerequisites

- Convex account (https://dashboard.convex.dev)
- Google Cloud Console project with OAuth 2.0 credentials
- Apple Developer account with Sign in with Apple configured
- SMS provider account (e.g., Twilio) for Phone OTP

### convex/auth.ts
```typescript
import { createClient, type GenericCtx } from "@convex-dev/better-auth";
import { convex } from "@convex-dev/better-auth/plugins";
import { betterAuth } from "better-auth/minimal";
import { expo } from "@better-auth/expo";
import { phoneNumber } from "better-auth/plugins";
import { components } from "./_generated/api";
import { DataModel } from "./_generated/dataModel";
import { query } from "./_generated/server";
import authConfig from "./auth.config";

export const authComponent = createClient<DataModel>(components.betterAuth);

export const createAuth = (ctx: GenericCtx<DataModel>) => {
  return betterAuth({
    trustedOrigins: ["nomadsafe://"],
    database: authComponent.adapter(ctx),
    socialProviders: {
      google: {
        clientId: process.env.GOOGLE_CLIENT_ID!,
        clientSecret: process.env.GOOGLE_CLIENT_SECRET!,
      },
      apple: {
        clientId: process.env.APPLE_CLIENT_ID!,
        clientSecret: process.env.APPLE_CLIENT_SECRET!,
      },
    },
    plugins: [
      expo(),
      convex({ authConfig }),
      phoneNumber({
        otpLength: 6,
        expiresIn: 300,
        sendOTP: async ({ phoneNumber: phone, code }, ctx) => {
          // TODO: Integrate SMS provider (Twilio, etc.)
          // For dev: log OTP to console
          console.log(`OTP for ${phone}: ${code}`);
        },
        signUpOnVerification: {
          getTempEmail: (phone) => `${phone}@phone.nomadsafe.app`,
          getTempName: (phone) => phone,
        },
      }),
    ],
  });
};

export const getCurrentUser = query({
  args: {},
  handler: async (ctx) => {
    return authComponent.getAuthUser(ctx);
  },
});
```

### Auth Client Usage Patterns

**Google Sign-In:**
```typescript
await authClient.signIn.social({ provider: "google" });
// On iOS with expo-auth-session, use idToken approach:
await authClient.signIn.social({
  provider: "google",
  idToken: { token: googleIdToken, accessToken: googleAccessToken },
});
```

**Apple Sign-In (native via expo-apple-authentication):**
```typescript
import * as AppleAuthentication from "expo-apple-authentication";

const credential = await AppleAuthentication.signInAsync({
  requestedScopes: [
    AppleAuthentication.AppleAuthenticationScope.FULL_NAME,
    AppleAuthentication.AppleAuthenticationScope.EMAIL,
  ],
});

await authClient.signIn.social({
  provider: "apple",
  idToken: { token: credential.identityToken! },
});
```

**Phone OTP:**
```typescript
// Send OTP
await authClient.phoneNumber.sendOtp({ phoneNumber: "+1234567890" });

// Verify OTP (signs in or creates account)
await authClient.phoneNumber.verify({
  phoneNumber: "+1234567890",
  code: "123456",
});
```

**Session Check:**
```typescript
const session = authClient.useSession();
// session.data — current session or null
// session.isPending — loading state
```

## Step 8: Biometric / PIN Local Auth

### services/secureStorage.ts

Wraps react-native-keychain for storing the PIN hash.

```typescript
import * as Keychain from 'react-native-keychain';

const PIN_SERVICE = 'nomadsafe-pin';

export const secureStorage = {
  async setPin(hashedPin: string): Promise<void> {
    await Keychain.setGenericPassword('pin', hashedPin, { service: PIN_SERVICE });
  },

  async getPin(): Promise<string | null> {
    const result = await Keychain.getGenericPassword({ service: PIN_SERVICE });
    return result ? result.password : null;
  },

  async hasPin(): Promise<boolean> {
    const result = await Keychain.getGenericPassword({ service: PIN_SERVICE });
    return !!result;
  },

  async resetPin(): Promise<void> {
    await Keychain.resetGenericPassword({ service: PIN_SERVICE });
  },
};
```

### utils/crypto.ts

PIN hashing using expo-crypto (SHA-256).

```typescript
import * as Crypto from 'expo-crypto';

export async function hashPin(pin: string): Promise<string> {
  return await Crypto.digestStringAsync(
    Crypto.CryptoDigestAlgorithm.SHA256,
    pin + 'nomadsafe-salt' // Static salt; PIN is already constrained to 6 digits
  );
}

export async function verifyPin(pin: string, storedHash: string): Promise<boolean> {
  const hash = await hashPin(pin);
  return hash === storedHash;
}
```

### services/localAuth.ts

Biometric authentication via expo-local-authentication.

```typescript
import * as LocalAuthentication from 'expo-local-authentication';

export const localAuth = {
  async checkBiometricAvailability(): Promise<{
    available: boolean;
    types: LocalAuthentication.AuthenticationType[];
  }> {
    const compatible = await LocalAuthentication.hasHardwareAsync();
    const enrolled = await LocalAuthentication.isEnrolledAsync();
    const types = await LocalAuthentication.supportedAuthenticationTypesAsync();
    return { available: compatible && enrolled, types };
  },

  async authenticateWithBiometric(): Promise<boolean> {
    const result = await LocalAuthentication.authenticateAsync({
      promptMessage: 'Unlock NomadSafe',
      cancelLabel: 'Use PIN',
      disableDeviceFallback: true, // We handle PIN fallback ourselves
    });
    return result.success;
  },
};
```

## Step 9: Theme Provider

### providers/ThemeProvider.tsx

```typescript
import React, { createContext, useContext, useMemo } from 'react';
import { useColorScheme } from 'react-native';
import { useSettingsStore } from '@/stores/settingsStore';
import { lightColors, darkColors, type ThemeColors } from '@/constants/theme';

interface ThemeContextValue {
  colors: ThemeColors;
  isDark: boolean;
}

const ThemeContext = createContext<ThemeContextValue | null>(null);

export function ThemeProvider({ children }: { children: React.ReactNode }) {
  const themeMode = useSettingsStore((s) => s.themeMode);
  const systemScheme = useColorScheme();

  const resolved = useMemo(() => {
    const effectiveScheme = themeMode === 'system' ? systemScheme : themeMode;
    const isDark = effectiveScheme === 'dark';
    return { colors: isDark ? darkColors : lightColors, isDark };
  }, [themeMode, systemScheme]);

  return (
    <ThemeContext.Provider value={resolved}>
      {children}
    </ThemeContext.Provider>
  );
}

export function useThemeContext() {
  const ctx = useContext(ThemeContext);
  if (!ctx) throw new Error('useThemeContext must be used within ThemeProvider');
  return ctx;
}
```

### app/index.tsx (Entry Redirect)

```typescript
// Determines which flow to show:
// 1. Not completed onboarding → (onboarding)/welcome
// 2. Not signed in → (auth)/sign-in
// 3. Signed in but not unlocked → (auth)/lock-screen
// 4. All good → (tabs)

import { Redirect } from 'expo-router';
import { useSettingsStore } from '@/stores/settingsStore';
import { useAuthStore } from '@/stores/authStore';

export default function Index() {
  const onboardingCompleted = useSettingsStore((s) => s.onboardingCompleted);
  const isSignedIn = useAuthStore((s) => s.isSignedIn);
  const isUnlocked = useAuthStore((s) => s.isUnlocked);
  const isPinSet = useAuthStore((s) => s.isPinSet);

  if (!onboardingCompleted) return <Redirect href="/(onboarding)/welcome" />;
  if (!isSignedIn) return <Redirect href="/(auth)/sign-in" />;
  if (isPinSet && !isUnlocked) return <Redirect href="/(auth)/lock-screen" />;
  return <Redirect href="/(tabs)" />;
}
```

### app/(tabs)/_layout.tsx (Tab Navigator)

5 tabs: Home, Trips, Expenses, Safety, Settings.
Uses Ionicons from `@expo/vector-icons`.
Tab bar styled with theme colors (tabBar, tabBarBorder, primary for active tint).

### app/(onboarding)/welcome.tsx

3-page horizontal FlatList with `pagingEnabled`:
1. "Track expenses, split costs" — expense icon illustration
2. "Stay safe with SOS alerts" — shield icon illustration
3. "AI-powered travel insights" — brain icon + "Get Started" button

On completion: `setOnboardingCompleted(true)` → navigate to sign-in.

### app/(auth)/sign-in.tsx

Three sign-in options:
1. "Continue with Google" button → `authClient.signIn.social({ provider: "google" })`
2. "Continue with Apple" button → native Apple auth → `authClient.signIn.social({ provider: "apple", idToken: {...} })`
3. Phone number input + "Send OTP" → navigate to phone-verify.tsx

After successful sign-in:
- If `!isPinSet` → navigate to setup-pin.tsx
- If `isPinSet` → `setUnlocked(true)` → navigate to (tabs)

### app/(auth)/phone-verify.tsx

6-digit OTP input (auto-focus, auto-advance between digits).
"Verify" button → `authClient.phoneNumber.verify({ phoneNumber, code })`.
"Resend" link with 60s cooldown timer.

### app/(auth)/setup-pin.tsx

Two steps:
1. "Create a 6-digit PIN" — numeric pad, 6 dots indicator
2. "Confirm your PIN" — re-enter to verify match

On confirm:
- Hash PIN via `hashPin()`
- Store hash via `secureStorage.setPin(hash)`
- `setPinSet(true)`
- Prompt biometric enrollment: "Enable Face ID/fingerprint?" → `setBiometricEnabled(true/false)`
- `setUnlocked(true)` → navigate to (tabs)

### app/(auth)/lock-screen.tsx

On mount:
- If `biometricEnabled`, auto-trigger `localAuth.authenticateWithBiometric()`
- Success → `setUnlocked(true)` → navigate to (tabs)

Fallback: 6-digit PIN pad.
- On submit: `verifyPin(input, storedHash)`
- Success → `setUnlocked(true)`
- Failure → shake animation, "Incorrect PIN" error
- 5 failures → 30s lockout (store failure count in memory, not persisted)

### app/(tabs)/settings.tsx

Settings sections:
- **Appearance**: Theme toggle (Light / Dark / System)
- **Security**: Change PIN, Toggle biometric, Auto-lock timeout (Immediate / 1 min / 5 min)
- **Account**: Name, email/phone display, Sign out button
- **About**: App version

## Step 12: App State Locking

In the root layout, add an AppState listener:

```
AppState change to 'background' → authStore.updateLastActive()
AppState change to 'active' →
  if (Date.now() - lastActiveTimestamp > autoLockTimeout)
    → authStore.setUnlocked(false)
    → router.replace('/(auth)/lock-screen')
```

This ensures the app locks after the configured timeout when returning from background.

This bridges the Convex real-time layer with local Zustand stores, so the UI always reads from Zustand (fast, offline-available) while Convex keeps it updated.
